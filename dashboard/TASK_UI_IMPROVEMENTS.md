# การปรับปรุง UI ของงานในแดชบอร์ด

## สรุปการปรับปรุง

### 1. เพิ่มปุ่มส่งงาน

**หน้างาน (Tasks)**
- เพิ่มปุ่ม "ส่งงาน" สำหรับงานที่มีสถานะ pending
- ปุ่มแสดงด้วยไอคอน paper-plane และสี warning (ส้ม)

**หน้าแดชบอร์ด (Dashboard)**
- เพิ่มปุ่ม "ส่งงาน" ในส่วน upcoming tasks
- ปุ่มแสดงเฉพาะงานที่มีสถานะ pending

### 2. ปรับปรุงการแสดงข้อมูลงานให้กระชับ

**หน้างาน (Tasks)**
- ลดพื้นที่การแสดงผลโดยรวมข้อมูลที่ไม่จำเป็น
- ย้ายความสำคัญของงานเป็นข้อความปกติไว้ใต้ชื่องาน
- รวมข้อมูลผู้รับผิดชอบ, วันที่ครบกำหนด, และสถานะในแถวเดียวกัน
- รวมข้อมูลโปรเจกต์, แท็ก, ไฟล์แนบ, และความคิดเห็นในแถวที่สอง (ถ้ามี)
- ลบข้อมูลที่ไม่จำเป็น: วันที่เริ่ม, วันที่เสร็จ, วันที่สร้าง, วันที่อัปเดต

**หน้าแดชบอร์ด (Dashboard)**
- ปรับปรุงการแสดงผลให้กระชับขึ้น
- ลดความยาวของ description จาก 100 เป็น 80 ตัวอักษร
- รวมข้อมูลในรูปแบบแถวเดียว

## การเปลี่ยนแปลงที่ทำ

### 1. view-renderer.js

#### หน้างาน (renderTasksList)
- เปลี่ยนจาก `.task-priority` badge เป็น `.task-priority-text` ข้อความปกติ
- เปลี่ยนจาก `.task-meta` เป็น `.task-meta-compact` พร้อม `.task-meta-row`
- รวมข้อมูลในแถวเดียวกัน: ผู้รับผิดชอบ, วันที่ครบกำหนด, สถานะ
- รวมข้อมูลเพิ่มเติมในแถวที่สอง: โปรเจกต์, แท็ก (แสดงแค่ 2 แท็กแรก), ไฟล์แนบ, ความคิดเห็น
- เพิ่มปุ่ม "ส่งงาน" สำหรับงาน pending

#### หน้าแดชบอร์ด (renderUpcomingTasks)
- เปลี่ยนจาก badges เป็นข้อความปกติ
- ใช้ `.task-meta-compact` และ `.task-meta-row`
- ลดความยาว description
- เพิ่มปุ่ม "ส่งงาน" สำหรับงาน pending

### 2. dashboard-core.js

#### เพิ่มฟังก์ชันใหม่
- `submitTask(taskId)`: จัดการการส่งงาน
- `showSubmitTaskModal(taskId)`: แสดง modal ส่งงาน
- `submitTaskToAPI(taskId, comment, files)`: ส่งข้อมูลไปยัง API

#### Modal ส่งงาน
- ฟอร์มสำหรับใส่ความคิดเห็น (ไม่บังคับ)
- อัปโหลดไฟล์แนบ (ไม่บังคับ, รองรับหลายไฟล์)
- ปุ่มยกเลิกและส่งงาน
- การจัดการ error และ loading

### 3. CSS

#### task-card.css
- เพิ่ม `.task-priority-text`: ข้อความความสำคัญแบบปกติ
- เพิ่ม `.task-meta-compact`: layout แบบกระชับ
- เพิ่ม `.task-meta-row`: แถวข้อมูล
- ปรับแต่งสีและขนาดของ elements ต่างๆ

#### components.css
- เพิ่ม CSS สำหรับ modal ส่งงาน
- เพิ่ม CSS สำหรับฟอร์มส่งงาน
- เพิ่ม `.btn-warning`: ปุ่มสีส้มสำหรับส่งงาน
- เพิ่ม CSS สำหรับ task-item ในหน้าแดชบอร์ด

## วิธีการทำงาน

### การส่งงาน
1. ผู้ใช้คลิกปุ่ม "ส่งงาน"
2. ระบบแสดง modal ส่งงาน
3. ผู้ใช้ใส่ความคิดเห็นและ/หรืออัปโหลดไฟล์
4. ระบบส่งข้อมูลไปยัง API `/api/groups/{groupId}/tasks/{taskId}/submit`
5. แสดงข้อความสำเร็จและรีเฟรชข้อมูล

### การแสดงผลงานแบบกระชับ
- ข้อมูลหลัก: ชื่องาน, ความสำคัญ, ผู้รับผิดชอบ, วันที่ครบกำหนด, สถานะ
- ข้อมูลเพิ่มเติม: โปรเจกต์, แท็ก, ไฟล์แนบ, ความคิดเห็น (แสดงเฉพาะถ้ามี)
- ลบข้อมูลที่ไม่จำเป็นออกเพื่อประหยัดพื้นที่

## ผลลัพธ์

### ✅ หน้างาน
- การแสดงผลกระชับขึ้น ใช้พื้นที่น้อยลง
- ข้อมูลสำคัญยังคงครบถ้วน
- มีปุ่มส่งงานสำหรับงานที่ยังไม่เสร็จ
- UI สะอาดและใช้งานง่าย

### ✅ หน้าแดชบอร์ด
- การแสดงผลกระชับขึ้น
- มีปุ่มส่งงานสำหรับงานที่กำลังจะถึงกำหนด
- ข้อมูลครบถ้วนในพื้นที่จำกัด

### ✅ ฟีเจอร์ส่งงาน
- Modal ส่งงานที่ใช้งานง่าย
- รองรับความคิดเห็นและไฟล์แนบ
- การจัดการ error ที่ดี
- UI ที่สวยงามและใช้งานง่าย
